## Code review of GagMate Android app (Gaggiuino espresso machine controller)
- Scanned full `app/src/main` (54 kt files, ~7k LOC). Wrote review to `/Users/yanx/Documents/GagMate/CODE_REVIEW.md`.
- **Critical:** `AppContainer.init()` reads `machineSession` (lateinit) as an arg to `SyncManager(...)` before assigning it → `UninitializedPropertyAccessException` at app launch. Fix: create `machineSession` before `syncManager`.
- **High:** `GaggiuinoV3Client.kt` (337 LOC) is dead code that duplicates `MachineSessionManager`/`ProtoDecoder`; its `handleDActProf`/`updateActiveProfile` are broken. WS host never refreshes after Settings change (only REST URL updates).
- **Medium:** `SyncManager.syncShots()` loops `1..latestShotId` with one sequential HTTP call per shot; `GgboardApiClient` debug log uses escaped `$`; WS host extraction only strips `http://`; `onClosed` doesn't reconnect; reconnect leaks OkHttpClient; `getConnectionUrl()` returns hardcoded default.
- Architecture is otherwise clean (Room + Retrofit + WS + Compose, Flow-based repos, CrashLogger). User only asked for review, not fixes.
