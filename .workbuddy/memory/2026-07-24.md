## Bug 修复：曲线全直线 + profile 详情无数据

用户附 `gagmate_combined(4).log`（REST-only）复核，反馈：① profile 详情仍无数据/无曲线；② 仪表盘"当前曲线"画出了但全为直线（未按 EASE_IN_OUT 缓动）。

根因：WS `d_prof`/`d_act_prof` protobuf **不携带 curve 类型**，`ProtoDecoder.decodePhaseInfo` 读 curve 为 varint 枚举（固件发 0）→ 恒 `"FLAT"`；真实 curve 字符串只在 REST `GET /api/profile/{id}` 与 shot 内嵌 `profile.phases`。旧 `fetchProfilePhases` 优先返回 WS 值 → 直线；且 `d_prof` name 提取失败时 deferred 超时 → 详情取空。

修复（已 `assembleDebug` 绿）：
- `MachineRepository.fetchProfilePhases` 重构为「WS 实时值 + 叠加 curve 来源」：WS 取实时值；另取 REST detail（若可用）或 shot 内嵌（按 name 或 `profile.id` 匹配）的真实 curve；phase 数一致时按序 overlay（`wp.copy(variation=cv)`）→ 缓动；否则回退 curve 来源/WS。
- `MachineSessionManager` WS `d_prof` 容错 name 匹配（exact 优先；仅 1 个 pending 时完成它）→ 修复详情页因 name 缺失超时取空。
- `ProtoDecoder.decodePhaseInfo` 增加字符串 curve 分支 + `normalizeCurveName`（防御）。
- `ProtoMessage.ActiveProfileMsg` 增加 `isActiveProfile` 标志。
- 文档同步：GAGMATE_REFERENCE.md 修正「REST `/api/profile/{id}` 不支持」等旧断言，CODE_REVIEW.md 新增 §0f。

注意：离线 `phasesJson`（来自 WS g_prof）curve 仍 `FLAT`；联网时由 `fetchProfilePhases` 叠加真实 curve。
